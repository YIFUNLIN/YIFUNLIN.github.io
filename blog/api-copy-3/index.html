API 深度解析：從核心概念到 Spring Boot 實戰

By Evan Lin 03 July 2025

今天要分享的是我先前整理的 API 深度解析文章，從核心概念講起，深入到不同的架構風格、安全性、設計實務，並以 Spring Boot 和 Kotlin 為例，提供一份從零開始建構與呼叫 API 的完整實戰指南，希望能幫助大家更系統性地理解 API。

一、API 到底是什麼？

想像一下您在餐廳用餐的場景：

  • 您 (Client)：想要點一份牛排。
  • 廚房 (Server)：負責烹飪牛排。
  • 服務生 (API)：您將需求告訴服務生，服務生將訂單傳達給廚房，最後再將烹飪好的牛排送到您的餐桌上。

在這個比喻中，您不需要知道廚房的內部運作（如何採購、烹飪），只需要透過「服務生」這個標準化的介面來取得您想要的「服務」（牛排）。

API 正是軟體世界中的「服務生」。它定義了不同軟體元件之間如何溝通、交換資料的規則與協定，讓開發者可以取用他人開發好的功能，而無需關心其內部複雜的實作細節。
